## 5. DESIGN RATIONALE

### 5.1 Three-View Architecture

**Decision**: Use view tabs (Timeline, By Source, By Story) instead of combined filter panels.

**Rationale**:
- Each view serves a different mental model
- Timeline = chronological discovery
- By Source = tool-centric workflow
- By Story = goal-oriented narrative building
- Reduces cognitive load by showing context-appropriate filters

### 5.2 Tree Lines Visualization

**Decision**: Draw SVG lines connecting filters to matching entries.

**Rationale**:
- Visual reinforcement of filter relationship
- Helps users understand groupings
- Delight factor - unexpected, memorable interaction
- Inspired by mind-mapping tools

### 5.3 Story Badges on Entries

**Decision**: Show clickable badges linking entries to their stories.

**Rationale**:
- Surfaces story connections in activity stream
- Enables quick navigation to story context
- Visual indicator of "linked" vs "unassigned" entries
- Supports the "evidence-backed" narrative concept

### 5.4 Visibility Selector Design

**Decision**: Use radio-style selection with icons and descriptions.

**Rationale**:
- Makes privacy implications clear
- Icons (ğŸ”’ ğŸ¢ ğŸŒ) provide instant recognition
- Descriptions prevent accidental over-sharing
- Matches mental model of privacy levels

### 5.5 Progressive Disclosure for Publishing

**Decision**: Require complete STAR before enabling publish, with clear guidance.

**Rationale**:
- Quality gate ensures profile has polished content
- "What's Missing" section guides completion
- Prevents embarrassing incomplete stories on profile
- Maintains trust in "evidence-backed" promise

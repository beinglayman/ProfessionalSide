Key Research Findings
  PLAN UPDATES FROM RESEARCH

  ✓ Pages Functions: file-based routing, not catch-all [[path]].ts
     functions/p/[shortCode].ts + functions/[slug].ts

  ✓ Use HTMLRewriter (streaming), not string .replace()
     Built into Workers runtime, handles edge cases

  ✓ Add public/_routes.json — excludes static assets from Functions
     Without it, every JS/CSS/image request burns free quota

  ✓ Fly.io: 512MB memory (Node+Prisma+tsx OOMs on 256MB)
     fly.toml lives in backend/fly.toml, not project root

  ✓ Neon: Prisma directUrl required in schema.prisma
     PgBouncer doesn't support advisory locks for migrations

  ✓ GitHub OAuth: dual-app strategy (only 1 callback URL per app)
     Create 2nd app for new domain, delete old after 2 weeks

  ✓ Razorpay webhook URL must update at cutover
     POST /api/v1/billing/webhook — missed in original plan

  ✓ Microsoft OAuth: add URIs 24hrs before cutover
     Propagation can take up to 24 hours

  ✓ Separate NS migration from traffic cutover (1 week apart)
     Reduces risk: verify email + site work through CF proxy first

  ✓ No PostgreSQL extensions used — clean Neon migration
     No cron jobs reference external URLs — no hidden dependencies
